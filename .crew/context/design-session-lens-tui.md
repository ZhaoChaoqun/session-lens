# 设计方案：Session Lens — 终端内会话浏览器

## 需求理解

- **用户场景**：Claude Code 用户在终端中工作，想快速回顾历史对话——"昨天那个 bug 我是怎么让 Claude 修的？"、"上周那个项目我聊到哪了？"
- **解决的问题**：Claude Code 没有 ChatGPT/Gemini 式的侧边栏，用户无法直观地浏览和检索历史会话
- **核心体验**：秒开，看一眼就走。像 `fzf` 一样——启动即可用，找到即退出

---

## 设计哲学

> "好的设计是尽可能少的设计。" — Dieter Rams

这个工具不是一个"会话管理系统"，而是一个**会话镜头**（Session Lens）——对准、聚焦、看清、离开。

三条设计红线：
1. **不超过 3 秒从启动到找到目标会话**
2. **不需要学习——第一次打开就知道怎么用**
3. **不打断工作流——随时进出，无状态残留**

参考 lazygit 的分栏范式，但更简洁。

```
├────────────────────────────┬────────────────────────────────────────┤

├────────────────────────────┤                                      │
│ ▸ Projects ▾               │   Session Preview                    │
│                            │                                      │
│   typeless (18)            │   ┌──────────────────────────────┐   │

│   Synapse (5)              │   │                              │   │

│  Mar 23  项目结构优化        │   │ 🤖 当前路径是                │   │

├────────────────────────────┤   Session: 5b0b0276...               │

└────────────────────────────┴────────────────────────────────────────┘
```

### 布局规格

| 区域 | 比例 | 最小宽度 |
|------|------|----------|
| 左栏（列表） | 35% | 28 字符 |
| 右栏（预览） | 65% | 40 字符 |

- **总最小宽度**：80 字符（低于此宽度切换为全屏列表模式，`Enter` 打开全屏预览）
- **全屏模式**：终端宽度 < 80 时，隐藏右栏，`Enter` 进入全屏预览，`Esc` 返回列表
- 分隔线使用 `│` 字符（Box Drawing），1 字符宽

### 左栏内部结构（从上到下）

1. **搜索栏**：1 行，顶部固定
2. **项目分组区**：可折叠的项目列表（当选中某项目时展开其会话列表）
3. **会话列表区**：选中项目后显示其下的会话，按时间倒序
4. **状态栏**：1 行，底部固定，显示统计信息

### 右栏内部结构

1. **预览内容区**：会话的消息流，可滚动
2. **会话元信息栏**：1 行，底部固定

## 二、会话列表信息展示

### 项目分组行

```
  typeless (18)                     ← 折叠态：项目名 + 会话数
▸ typeless (18)                     ← 展开态：带展开指示符
```

项目名从目录名提取，规则：
- `-Users-zhaochaoqun-Github-typeless` → `typeless`
- `-Users-zhaochaoqun-Github-claude-web-chat` → `claude-web-chat`
- `-Users-zhaochaoqun` → `~ (home)`
- 去掉 `--crew-roles-*` 后缀的子工作区会话

### 会话行

```
  Mar 04  Rust修复了长音频截断问题，请重新运行benchmark
  │       │
  │       └── 第一条用户消息的前 N 个字符（N 随列宽自适应）
  └────────── 日期（同年省略年份，跨年显示年份）
```

**显示信息**：
| 字段 | 来源 | 显示规则 |
|------|------|----------|
| 日期 | 第一条记录的 `timestamp` | `MMM DD` 格式（如 `Mar 04`），跨年加年份 |
| 摘要 | 第一条非 meta 的 `type=user` 消息的 `message.content` | 截断到列宽-10 字符，去除 XML 标签 |
| 当前选中行高亮 | — | 反色（reverse video）或蓝色背景 |

**不显示**（减噪）：
- Session ID（UUID 对用户毫无意义）
- 文件大小（不是用户关心的维度）
- 消息数量（在底部状态栏按需显示）

### 排序逻辑

| 层级 | 排序规则 |
|------|----------|
| 项目 | 按最近会话时间倒序（最近活跃的项目在最上面） |
| 会话 | 按文件修改时间倒序（最新会话在最上面） |

## 三、会话预览交互

### 预览内容格式

右栏显示选中会话的消息流，只显示**用户消息和助手的文本回复**，过滤掉：
- `tool_use` / `tool_result` 块（噪音太大）
- `progress` 类型记录
- `queue-operation` 类型记录
- `file-history-snapshot` 类型记录
- `isMeta: true` 的消息
- `system` 类型的技术消息

### 消息渲染格式

```
 ▌ Rust修复了长音频截断问题，请重新运行一次benchmark
 ▌ 跑一下所有的pipeline最新的结果

   我先了解一下当前的 benchmark 测试文件和构建状态。

 ▌ 我理解是因为rust的更新在运行benchmark前
 ▌ 没有推送到远端，我刚才已经推送了

   好的，让我重新运行 benchmark。

 ▌ 目前Swift中使用的一些超参数是如何设置的

   让我查看当前的参数配置...
```

- 用户消息：左侧带 `▌` 竖线标记（使用主题色，如蓝色/青色），文本颜色正常
- 助手消息：无标记，文本前缩进 3 空格，颜色略暗（dim）
- 消息之间：空 1 行分隔
- 长消息：自动换行（word wrap），不截断

### 预览滚动

- 默认显示会话**开头**（用户最可能想看"这个会话聊的什么"）
- `j/k` 或 `↑/↓` 在列表中切换会话时，预览自动刷新到顶部
- `Tab` 将焦点从左栏切换到右栏，此时 `j/k` 滚动预览内容
- `Shift+Tab` 或 `Tab` 将焦点切回左栏
- `G` 跳到预览底部（看最近的对话）
- `g` 跳到预览顶部

### 大文件性能策略

- **懒加载**：预览只解析前 500 行 JSONL（覆盖大部分浏览需求）
- **按需加载更多**：滚动到底部时自动加载下一批 500 行
- **索引缓存**：首次扫描时将每个会话的元信息（首条消息、时间、消息数）缓存到内存

## 四、搜索/过滤交互

### 搜索栏

```
┌────────────────────────────┐
│ 🔍 benchmark typeless      │   ← 实时过滤模式
└────────────────────────────┘
```

### 搜索行为

1. **按 `/` 或直接输入**激活搜索栏（焦点在搜索栏时表现为 fzf 模式）
2. **实时模糊过滤**：每输入一个字符，立即过滤会话列表
3. **搜索范围**：
   - 项目名 
   - 会话首条用户消息（已加载的摘要文本）
   - 不做全文搜索（性能考量，JSONL 文件太大）
4. **`Esc`** 清除搜索并退出搜索模式
5. **`Enter`** 确认搜索结果，选中当前高亮的会话
6. 搜索结果自动压平项目分组——直接显示匹配的会话行，前缀加项目名

### 搜索结果行格式

```
  typeless  Mar 04  Rust修复了长音频截断问题...
  typeless  Mar 11  以下参数都使用默认值...
  ai-pr     Mar 16  review PR #42 的变更...
```

### 项目过滤快捷方式

- `p` 键打开项目过滤面板（弹出一个浮动选择框）
- 可多选项目进行过滤

## 五、核心操作流程

### 流程：浏览 → 预览 → 恢复

```
启动 session-lens
       │
       ▼
  ┌──────────────┐    直接看到按时间排序的会话列表
  │  浏览列表     │    ← 光标默认在最新会话上
  │  (j/k 上下)  │
  └──────┬───────┘
         │
    ┌────┴────┐
    │ 右栏预览  │    ← 选中即预览（跟随光标自动刷新）
    │ (自动)   │
    └────┬────┘
         │
         ├── Enter: 用 claude --resume 恢复会话 ──→ 退出工具，在当前终端恢复会话
         │
         ├── /: 搜索过滤 ──→ 缩小范围 ──→ 回到浏览
         │
         ├── Tab: 进入预览滚动模式 ──→ 详细阅读 ──→ Tab 回到列表
         │
         ├── y: 复制 session ID 到剪贴板
         │
         └── q/Esc: 退出工具
```

### 启动方式

```bash
session-lens              # 直接启动，显示所有项目的会话
session-lens typeless     # 只显示 typeless 项目的会话
session-lens --search "benchmark"  # 启动即搜索
```

### 恢复会话

按 `Enter` 后：
1. 工具退出
2. 在当前终端执行 `claude --resume <session-id>`（输出到 stdout，用户在原终端中继续）
3. 如果用户在特定项目目录下启动 session-lens，但选中的会话属于另一个项目，提示：  
   `⚠ This session belongs to project "typeless". Switch directory? (y/n)`

## 七、状态设计

### 默认态（启动后）
- 左栏显示项目列表，按最近活跃排序
- 最近的项目自动展开，光标在第一个会话上
- 右栏显示该会话的预览

### 空状态
- 如果 `~/.claude/projects/` 目录不存在或为空：
  ```
  ┌──────────────────────────────────────────┐
  │                                          │
  │   No sessions found.                     │
  │                                          │
  │   Start a Claude Code session first:     │
  │   $ claude                               │
  │                                          │
  └──────────────────────────────────────────┘
  ```

### 搜索无结果
```
  🔍 xyzabc123
  
  No matching sessions.
  
  Press Esc to clear search.
```

### 加载中（大目录首次扫描）
- 启动时显示简短的 spinner：`Scanning sessions...`
- 扫描完成后立即切换到列表（目标：< 500ms）
- 后续启动使用缓存，瞬时显示

### 删除确认
```
┌──────────────────────────────────────┐
│ Delete this session?                 │
│                                      │
│ Mar 04 · Rust修复了长音频截断问题...    │

│                                      │
```

浮动居中弹窗，`y` 确认删除，`n`/`Esc` 取消。

## 八、视觉规格

### 配色方案

采用终端原生 ANSI 256 色，自动适配亮色/暗色终端：

| 元素 | 暗色终端 | 说明 |
|------|----------|------|
| 背景 | 终端默认背景 | 不强制背景色，尊重用户终端主题 |
| 普通文本 | 终端默认前景 | — |
| 选中行 | 蓝色背景 `\e[44m` + 白色文字 | 高对比高亮 |
| 用户消息标记 `▌` | 青色 `\e[36m` | 视觉锚点 |
| 助手消息 | dim `\e[2m` | 降低视觉权重 |
| 项目名 | 粗体 `\e[1m` | 分组标题 |
| 日期 | dim `\e[2m` | 辅助信息，不抢眼 |
| 搜索匹配字符 | 黄色 `\e[33m` + 粗体 | 高亮匹配部分 |
| 分隔线 | dim `\e[2m` | 最小视觉干扰 |
| 状态栏 | 反色 `\e[7m` | 固定在底部 |
| 帮助浮层 | 与选中行相同背景 | 一致性 |

### 字符与边框

- 分栏分隔：`│`（U+2502 Box Drawings Light Vertical）
- 项目展开指示：`▸`（折叠）/ `▾`（展开）
- 用户消息标记：`▌`（U+258C Left Half Block）
- 不使用粗边框或双线边框——最小视觉噪音

### 间距

| 元素 | 间距 |
|------|------|
| 搜索栏与列表 | 1 行分隔线 |
| 项目行之间 | 0 行（紧凑） |
| 会话行之间 | 0 行（紧凑） |
| 预览中消息之间 | 1 空行 |
| 左栏内容左缩进 | 2 空格 |
| 预览内容左缩进 | 2 空格 |
| 用户消息文本缩进 | `▌` 后 1 空格 |
| 助手消息文本缩进 | 3 空格 |

## 九、响应式适配

### 宽屏（≥ 120 字符）

标准双栏模式，预览区有充足空间显示长消息。

### 标准（80-119 字符）

双栏模式，左栏压缩到 28 字符最小宽度，摘要文本截断更多。

### 窄屏（< 80 字符）

切换为**单栏全屏模式**：
- 默认显示会话列表（全屏宽度）
- `Enter` 进入全屏预览
- `Esc` / `q` 从预览返回列表
- 体验类似手机端的推入/弹出导航

### 终端高度适配

- 最小高度：10 行（低于此高度只显示列表，不显示搜索栏和状态栏）
- 建议高度：24 行以上

## 十、边界场景

| 场景 | 处理方式 |
|------|----------|
| JSONL 文件损坏 | 跳过损坏行，继续解析，不报错 |
| 超大文件（>50MB） | 预览仅解析前 1000 条记录，底部提示 `[showing first 1000 messages]` |
| 会话无用户消息 | 摘要显示 `(empty session)`，dim 颜色 |
| 项目目录名超长 | 截断到列宽-5，末尾加 `…` |
| Unicode/CJK 文本 | 正确处理全角字符宽度（每个 CJK 字符占 2 列） |
| subagents 子目录 | 不在主列表显示，避免噪音 |
| 摘要中含 XML/HTML 标签 | 解析时去除 `<...>` 标签，只保留纯文本 |

## 十一、数据提取规格

### 会话元信息提取

从每个 JSONL 文件提取：

```typescript
interface SessionMeta {
  id: string;           // 文件名（UUID）
  project: string;      // 父目录名 → 可读项目名
  timestamp: Date;      // 文件 mtime 或首条记录的 timestamp
  summary: string;      // 第一条非 meta 的 user 消息文本，去除 XML 标签
  messageCount: number; // type=user 或 type=assistant 的记录数
  branch: string;       // 首条记录的 gitBranch 字段
  cwd: string;          // 首条记录的 cwd 字段
  fileSize: number;     // 文件字节大小
}
```

### 摘要提取逻辑

1. 逐行读取 JSONL
2. 找到第一条 `type === 'user' && !isMeta` 的记录
3. 提取 `message.content`：
   - 如果是 string：去除 XML 标签，取前 200 字符
   - 如果是 array：找第一个 `type === 'text'` 的元素，同上处理
4. 如果内容以 `<command-name>` 开头（如 `/cost`），跳过，继续找下一条
5. 找到后停止读取该文件（不需要读完整个文件）

### 性能优化

- **首屏加载**：只读取每个文件的前 50 行（足够提取元信息）
- **预览加载**：选中会话时读取前 500 行
- **并行扫描**：可并行读取多个 JSONL 文件的头部
- **文件系统缓存**：使用 `mtime` 判断是否需要重新解析
